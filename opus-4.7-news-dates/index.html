Does Opus 4.7 forget the news? An empirical investigation of a claim about Claude's latest model

By Tomasz Kolinko · April 17, 2026 · 44 blind-judged questions across 3 independent test batteries

A friend reached out after Anthropic released Claude Opus 4.7 (April 16, 2026) with a concerning observation.

"Ważne! Opus 4.7 nie ogarnia dat newsów i aktualizacji tak jak robił to 4.6. To jest core quality modelu dla wielu moich rzeczy więc sprawdziłem i nowy Opus spadł do jakości Sonneta w tej konkretnej kwestii 🫢. To jest fascynujące, ale jednocześnie to jest w moim przypadku red flag." Translation: "Important! Opus 4.7 doesn't handle news dates and updates as well as 4.6 did. This is core quality for many of my use cases — the new Opus dropped to Sonnet-level quality on this specific issue. Fascinating, but also a red flag for me."

This kind of claim is easy to assert and hard to verify — I run many things on Opus, and if 4.7 regressed on date awareness, it would be bad news. So I spent a few hours running controlled blind tests to find out.

The short answer: the opposite appears to be true. On all three test batteries, Opus 4.7 scored higher than Opus 4.6 on news dates, software update timelines, and chronological reasoning. On the specific kind of "what's the latest X?" questions that match the friend's described use case, 4.7 outperformed 4.6 by a wide margin.

Friend's claim: not supported by the data. Overall score (0-10, blind-judged): Opus 4.6 = 8.66 · Opus 4.7 = 9.23 · Sonnet 4.6 = 8.80 (n = 44 questions)

Methodology in one minute

How the tests work

  • Three independent question batteries, each probing a different angle of "news date awareness"
  • Every question asked to all three models via direct API (no tools, no web search, no system prompt — raw model knowledge)
  • An LLM judge scored each answer 0-10 against a known ground truth
  • Judge was blinded: answers labeled X/Y/Z, presentation order shuffled per test
  • Each model got exactly one attempt per question

The "what's latest" round is the one closest to what the friend described, and it's also where the gap is largest. Opus 4.7 scores 1.46 points higher than 4.6 on those 13 questions — 19% improvement.

The secondary claim that 4.7 "dropped to Sonnet level" is also not supported: 4.7 scored above Sonnet 4.6 in every round, with a pooled gap of 0.43 points.

Hallucinations: Opus 4.6 fabricated specific wrong details 2 times, Opus 4.7 did so 1 times, Sonnet 0 times. All three are in similar territory — no systematic hallucination increase in 4.7.

Where 4.7 loses or ties

To be fair, 4.7 doesn't beat 4.6 on every question. On all common, well-known events from 2024 (ChatGPT launch, GPT-4o release, Paris Olympics, Trump inauguration), the three models score identically — all getting the right answer confidently.

The only round-1 question where everyone struggled was a chronological ordering puzzle involving Gemini 1.5 Pro's general availability date, which is genuinely ambiguous (preview vs GA). All three models scored below 6.

Interesting observation from the "cursor-composer" case above: when the answer is genuinely uncertain, Sonnet 4.6 correctly hedged while both Opus versions confidently hallucinated the wrong month. That's a Sonnet win neither Opus could replicate. If the friend happened to hit that kind of question on 4.7, and got a confident wrong answer, the subjective impression of "quality drop" would be understandable — even though the overall numbers show 4.7 is more reliable on date facts.

What the friend might have actually observed

Claims about LLM quality are often shaped by a few memorable bad experiences. A few possibilities:

  • Small sample — 1-2 test queries can easily produce misleading impressions either direction
  • A specific domain-narrow blind spot this test battery doesn't cover
  • Interaction with a system prompt or tool use pattern that changes behavior
  • Confusion with a different model or the same model at a different time

If you have specific examples where 4.7 seems worse on dates, please send them — I'd happily add them to the test battery and see if the pattern replicates.

Takeaway

For the use case of asking Opus about news dates, software release timelines, and "what's the latest X" questions, Opus 4.7 is as good as or better than Opus 4.6. The quality-drop concern doesn't hold up under controlled testing.

If you're migrating from 4.6 to 4.7 and were worried about this specific regression — you don't need to be.
