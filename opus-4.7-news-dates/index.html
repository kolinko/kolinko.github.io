Methodology

How the tests work

  • Four independent test batteries: three for training knowledge, one for document parsing
  • Every question asked to all three models via direct API (no tools, no web search, no system prompt)
  • Document parsing uses: a real Hacker News front page (2025-01-20), a synthetic but realistic news listing, an email thread with relative dates, a Wikipedia AI timeline excerpt
  • Judge: Claude Opus, blinded. Each model's answer labeled X/Y/Z with presentation order shuffled per test. Judge prompt explicitly states "You do NOT know which model produced which output."
  • Blinding verified: scanned all outputs for identity leaks ("I am Claude", "Opus 4.X", "as Claude Sonnet", etc.) — 0 leaks across all ~165 model outputs
  • Position bias controlled: an earlier experiment (in the full 4.6 vs 4.7 comparison) quantified judge's first-position bias at ~+0.6 points — not enough to explain the 4.7 advantage
  • Each model got exactly one attempt per question. Single-shot, no retries.
