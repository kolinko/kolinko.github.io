Does Opus 4.7 forget the news? An empirical investigation of a claim about Claude's latest model

By Tomasz Kolinko · April 17, 2026 · 55 blind-judged questions across 4 independent test batteries

A friend reached out after Anthropic released Claude Opus 4.7 (April 16, 2026) with a concerning observation.

"Ważne! Opus 4.7 nie ogarnia dat newsów i aktualizacji tak jak robił to 4.6. To jest core quality modelu dla wielu moich rzeczy więc sprawdziłem i nowy Opus spadł do jakości Sonneta w tej konkretnej kwestii 🫢. To jest fascynujące, ale jednocześnie to jest w moim przypadku red flag." Translation: "Important! Opus 4.7 doesn't handle news dates and updates as well as 4.6 did. This is core quality for many of my use cases — the new Opus dropped to Sonnet-level quality on this specific issue. Fascinating, but also a red flag for me."

This kind of claim is easy to assert and hard to verify. If 4.7 regressed on date awareness, it would be bad news — many workflows depend on this kind of reliability. So I spent a few hours running controlled blind tests.

The tests cover two very different interpretations of the claim:

  • Training knowledge — "When was X released?" Can the model recall dates from its training?
  • Document parsing — "Given this news page, extract the dates." Can the model reason about dates in provided text?

On both interpretations, the answer is the same: the opposite of the friend's claim appears to be true. Opus 4.7 scored higher than Opus 4.6 on every single round.

Friend's claim: not supported by the data. Overall score (0-10, blind-judged): Opus 4.6 = 8.58 · Opus 4.7 = 9.22 · Sonnet 4.6 = 8.69 (n = 55 questions)

Methodology in one minute

How the tests work

  • Four independent question batteries: three probing training knowledge, one probing document-parsing
  • Every question asked to all three models via direct API (no tools, no web search, no system prompt — raw model behavior)
  • Document-parsing tests include a real Hacker News front page, a real Wikipedia article excerpt, a synthetic news listing, and an email thread with relative dates ("last Wednesday", "two weeks ago")
  • An LLM judge scored each answer 0-10 against a known ground truth
  • Judge was blinded: answers labeled X/Y/Z, presentation order shuffled per test
  • Each model got exactly one attempt per question

When Sonnet actually wins (the interesting exception)

In the cursor-composer case above, both Opus versions confidently hallucinated the wrong month (stated October 2024 — actual answer is November 2024). Only Sonnet correctly hedged, saying it wasn't sure.

This is a real failure mode that affects both Opus versions equally. If the friend encountered this kind of question on 4.7, and noticed a confident wrong answer, the subjective impression of regression would be understandable — even though the overall numbers show 4.7 is actually slightly more reliable than 4.6 on date facts.

What the friend might have actually observed

Claims about LLM quality are often shaped by a few memorable bad experiences. A few possibilities:

  • Small sample — 1-2 test queries can produce misleading impressions in either direction
  • A specific domain-narrow blind spot this test battery doesn't cover
  • Interaction with a system prompt or tool-use pattern that changes behavior
  • Confirmation bias after seeing one confident hallucination
  • Confusion with a different model version or different time window

If you have specific examples where 4.7 seems worse on dates, please send them — I'd happily add them to the test battery and see if the pattern replicates.

Takeaway

For the use case of asking Opus about news dates, software update timelines, or extracting dates from documents, Opus 4.7 is as good as or better than Opus 4.6. The quality-drop concern doesn't hold up under controlled testing.

If you're migrating from 4.6 to 4.7 and were worried about this specific regression — you don't need to be.
